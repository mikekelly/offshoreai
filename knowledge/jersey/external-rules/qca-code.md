# QCA Corporate Governance Code — Jersey Interaction

**Bridge file.** The Quoted Companies Alliance ("QCA") publishes
the **QCA Corporate Governance Code** — the **most commonly
adopted governance code among AIM-listed companies**, including
Jersey-incorporated AIM TopCos. For the substance of the Code
and any commentary, cite the current QCA publication directly.

## Why the QCA Code matters for Jersey AIM companies

[AIM Rule 26](./aim-rules.md) requires every AIM company to
disclose on its website **which corporate governance code it
follows** and **how it applies that code**. AIM does not mandate
which code; companies choose. The two practical choices are:

1. **The QCA Code** — designed specifically for growth-company
   AIM-listed issuers; proportionate to AIM company size and
   stage. **Most AIM companies adopt this.**
2. **The [UK Corporate Governance Code](./uk-corporate-governance-code.md)**
   — designed for premium-listed Main Market issuers;
   significantly more demanding; occasionally adopted by larger
   AIM companies whose investor base expects Main-Market-grade
   governance.

For a **Jersey-incorporated AIM TopCo**, the QCA Code is almost
always the right choice unless the company is unusually large or
has a Main-Market-style investor base.

## Apply and explain

The QCA Code operates on an **"apply and explain"** basis. The
company explains how it applies each of the Code's ten
principles to its specific circumstances. There is **no "comply
or explain"** — i.e., the company is not asked to identify
non-compliance with prescriptive rules; instead it describes its
own application of each principle.

This is more flexible than the UK Corporate Governance Code's
"comply or explain" structure and better-suited to AIM company
scale.

## The ten principles (summary)

The QCA Code is built around ten principles spanning:

1. Establishing a strategy and business model that promotes
   long-term value for shareholders.
2. Seeking to understand and meet shareholder needs and
   expectations.
3. Taking into account wider stakeholder and social
   responsibilities and their implications for long-term
   success.
4. Embedding effective risk management, considering both
   opportunities and threats, throughout the organisation.
5. Maintaining the board as a well-functioning, balanced team
   led by the chair.
6. Ensuring that between them the directors have the necessary
   up-to-date experience, skills and capabilities.
7. Evaluating board performance based on clear and relevant
   objectives, seeking continuous improvement.
8. Promoting a corporate culture that is based on ethical
   values and behaviours.
9. Maintaining governance structures and processes that are fit
   for purpose and support good decision-making by the board.
10. Communicating how the company is governed and is performing
    by maintaining a dialogue with shareholders and other
    relevant stakeholders.

Each principle is accompanied by **practical guidance** on
application and **disclosure expectations**.

## Interaction with Jersey CJL 1991

The QCA Code does **not displace** Jersey company law. A
Jersey-incorporated AIM company adopting the QCA Code remains
governed by **Article 74 CJL 1991** statutory and common-law
directors' duties (see
[`../companies/directors-duties.md`](../companies/directors-duties.md))
and by every other CJL 1991 provision.

The QCA Code is a **governance overlay**: it tells the company
how to **structure and disclose** its governance. It does not
change the underlying legal duties. Where there is any
inconsistency between the QCA Code and CJL 1991, CJL 1991 wins.

In practice, the principles align well — both regimes converge
on board accountability, risk management, and shareholder
engagement.

## AIM Rule 26 disclosure mechanism

The AIM Rule 26 website disclosure on governance is the
operational mechanism by which QCA Code adoption becomes
visible. The expected disclosure includes:

- **Statement of governance code adopted** (QCA, UKCGC, or
  other);
- **How the chosen code is applied** — typically as an
  "apply and explain" narrative against each principle for QCA
  adopters;
- **Departures or non-application** — for QCA Code adoption,
  this is generally addressed within the principle-by-principle
  application narrative rather than as a separate "non-
  compliance" section.
