# Use Cases — Jersey

Question-driven entry points into the Jersey corpus, organised by the
**kind of user** asking. Each persona's `index.md` lists the questions
that user actually has, and each question gets its own file that
answers it end-to-end with links into the [topic graph](../index.md)
for the underlying rules.

The use-cases layer is intentionally short and pragmatic: the
topic-graph files are where the statute and case-law citations live;
the use-case files are where the user is told **what to do**, **what
to ask next**, and **where to look** in the topic graph for the
authority.

Each persona's file set is designed so a chunky question of the form
*"as a [persona] doing [task], what do I need to know?"* lands in one
to three use-case files, plus the supporting topic-graph files
those use-cases link to.

## How a use-case file is structured

1. **The question**, stated in the user's voice (no formal legalese).
2. **The short answer** — the rule of thumb, in 2–4 sentences.
3. **The proper answer** — the rule, the test, the case-law, with
   inline citations. Statute references go to
   [jerseylaw.je](https://www.jerseylaw.je); regulator material goes
   to [jerseyfsc.org](https://www.jerseyfsc.org); Revenue Jersey
   material goes to its [gov.je tax pages](https://www.gov.je/TaxesMoney).
4. **What to do next** — the operational checklist.
5. **Pitfalls** — the things that catch people out.
6. **Related** — links to other use-cases and to the topic graph.
